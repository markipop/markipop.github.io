黑苹果更新opencore版本

黑苹果更新opencore版本

前言

名称 属性
笔记本型号 暗影精灵5
系统版本 Big sur 11.6
Opencore版本 0.6.3

今天偶然逛BobMaster’s Blog，看到了BobMaster更新的OC文件，在此之前我一直用的是他开放出来的0.6.3版本的EFI文件。这个版本伴随着我更新到了11.6最新的正式版，

平时使用一切ok，除了蓝牙共享有些问题（无法通过手机/ipad/电脑直接通过蓝牙分享文件），其他都很好。只是不知道更新到最新的OC版本是否会解决这个问题。

这篇笔记的前提是有了正常使用的苹果系统，需要做的只是替换OC相关的文件。

因为用的是双系统，所以既可以在windows下操作也可以在macOS下操作OC版本更迭，方便就好，我是在windows下整的。

使用DiskGenius来操作。

下载EFI文件

  • 从github.com下载最新版本的EFI文件

备份原EFI文件

备份十分重要！！！万一错误操作或者EFI不合适，可能直接让电脑重启后无法正常引导启动。

替换EFI

，确认备份好整个EFI文件后

  1. 找到系统引导文件存放的位置，删除OC文件以及Boot文件目录下的BOOTx64.efi文件。

  1. 将下载好的EFI文件解压缩，然后在EFI的目录下找到下图红框标注的部分。

  2. 将解压好的EFI文件中OC文件夹拖入第1步删除OC文件的位置，将BOOT下BOOTx64.efi拖入到Boot文件的目录内。

  3. 完成之后不要忘了查看报告，以确定转移过程没有出错。

重启

重启检查是否有错，如果出现了引导失败的情况就需要确认是否有将BOOTx64.efi文件替换成压缩包内的最新版本的文件，如果遇到了其他问题，欢迎留言。

重启正常的话，会进入和未更新前类似的文字界面，下一步以选择哪个系统。

有关更新0.7.4版本更新的内容，以及对应这一版本修改的内容，可以参考官网和BobMaster’s Blog

Opencore图形引导界面

因为上一步下载的EFI文件精简过，省去了不必要的一些东西，其中就包括图形引导界面。

我这里只添加了图形引导，至于开机音频就没有添加了，如果你也想要图形引导界面，接着往下看就好了。

编辑Opencore

可以用官方的编辑器，也可以用第三方的，建议还是用官方的，减小污染配置文件的可能性。我用的是OCAT，第三方的，链接可以百度，挺好找的，如果找不到，可以留言。

补充文件

  1. 将上一步正常跑的OC文件备份出来，方便操作，等所有东西弄完，直接覆盖引导分区内引导文件，简单粗暴。

  2. 去官网下载对应版本的完整的Opencore文件，因为我清楚我下载的是0.7.4版本，所以我也找了官方的对应完整版本。

  3. 解压官网下载的完整版的文件，从EFI\OC\Drivers\中找到OpenCanopy.efi文件，复制下来，粘贴到备份好的OC文件对应的目录下。

    注意：这里需要注意Opencanopy文件和opencore版本需要一致！

  4. 补充resourse文件夹的资源，这里使用lxyoucan分享的，https://pan.baidu.com/s/1qnnBumyc3ZanuI04OACasg 提取码: b7ps

  5. 接下来就简单了，下载好后吧对应的文件分别拖放覆盖

编辑config.plist文件

  1. 用OCAT打开备份出来的OC文件中config.plist文件
  2. 修改Misc -> Boot -> PickerMode为External
  3. 修改Misc -> Boot -> PickerAttributes为1
  4. UEFI->Drivers-> 添加OpenCanopy.efi（路径所在）

完善EFI文件

将备份的OC文件内ACPI、Drivers、Kexts分别拖放到新生成的EFI文件

替换EFI

这一步其实就是本篇第一节替换EFI的内容，这里就不赘述了，卡在细节问题时，解决不了可以留言在博客中

结语

回顾一下，这篇写了如何在现有ok的opencore文件下，更新旧版本的opencore文件，其次写了如何在精简版的opencore上增加图形引导界面。
